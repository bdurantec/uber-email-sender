# uber-email-service

## Notes

CleanArchitecture

* Core - reúne casos de uso e regras de negócio da aplicação. Define "o que" o app faz e não "como" faz.
* Application - implementa os casos de uso definidos na camada Core.
* Adapters - contrato da aplicação com o provedor e-mail utilizado, deixando o service independente da tecnologia
  utilizada.
* Infra - comunica-se com serviços externos.
